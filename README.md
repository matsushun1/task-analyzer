# task-analyzer

Notion で管理しているタスクとデイリーノートを分析し、Claude AI による日次レポートを自動生成する TypeScript バックエンドサービスです。

---

## 概要

毎日の朝、Notion のボタンを押すだけで、進行中のタスクの優先順位付け・期限切れタスクの洗い出し・体調やタスク管理へのアドバイスを含むレポートが自動生成されます。

処理フローは以下の通りです。

```
Notion ボタン → Notion Automations → Make.com → Cloud Run (本サービス) → Claude API
                                                                              ↓
                                                                       Notion レポートページ作成
```

---

## 対象ユーザー

- Notion でタスク管理とデイリーノートを運用している個人
- 日々のタスク優先度の整理や振り返りを効率化したい人

---

## 主な機能

| 機能 | 説明 |
|------|------|
| 最初の一手の提示 | 今日まず着手すべきタスクを1件選び、「PRのURLをSlackに貼る」などの物理的な最初の行動を提示 |
| 今日やるべきタスクの抽出 | Status="Doing" のタスクから期限・工数・デイリーノートを考慮し、Claude AI が最大 5 件を優先順位付き一覧で提示 |
| 期限切れタスクの検出 | 進行中だが期限が過去のタスクをすべて抽出して一覧表示 |
| 体調アドバイス | 直近 14 日分のデイリーノート【健康状態】セクションを分析し、改善提案を生成 (100-200 文字) |
| タスク管理アドバイス | 工数・課題セクションを分析し、具体的な次の一手を含むアドバイスを生成 (100-200 文字) |

---

生成されるレポートは Notion ページとして出力されます。

```

━━━━━━━━━━━━━━━━━━━━━━━━

## ⭐ 今日まず最初にやること

予約登録API開発
最初の一手: PRのURLをSlackに貼る

━━━━━━━━━━━━━━━━━━━━━━━━

## 🎯 今日やるべきタスク

1. 予約登録API開発 (期限: 3/28)
   理由: 昨日が期限。今日対応しないとリリースに影響します。

2. テスト仕様書作成 (期限: 3月下旬)
   理由: 月末まで残り2日。本日中に着手が必要です。

...（最大5件）

━━━━━━━━━━━━━━━━━━━━━━━━

## ⚠️ 期限切れタスク

• 複数同時予約同期問題対応 - 期限: 3/20
• ドキュメント整備 - 期限: 3/25

━━━━━━━━━━━━━━━━━━━━━━━━

## 💪 体調アドバイス

直近2週間で疲労感の記録が増加しています。今日は集中ブロックを
1回だけにして、無理のないペースで進めましょう。

━━━━━━━━━━━━━━━━━━━━━━━━

## 📝 タスク管理アドバイス

同じ課題が3日続いて記録されています。今日中に解決の目処が
立たなければ「○○の方針を確認したい」と上司に伝えましょう。

━━━━━━━━━━━━━━━━━━━━━━━━

生成日時: 2026/03/29 09:30
```

## 技術スタック

| 要素 | 技術 |
|------|------|
| 言語 | TypeScript (strict モード) |
| ランタイム | Node.js / Cloud Run |
| Notion 連携 | `@notionhq/client` |
| AI 分析 | Claude API (`claude-sonnet-4-6`) |
| テスト | Jest |
