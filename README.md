# task-analyzer

Notion で管理しているタスクとデイリーノートを分析し、Claude AI による日次レポートを自動生成する TypeScript バックエンドサービスです。

---

## 概要

毎日の朝、Notion のボタンを押すだけで、進行中のタスクの優先順位付け・期限切れタスクの洗い出し・体調やタスク管理へのアドバイスを含むレポートが自動生成されます。

処理フローは以下の通りです。

```
Notion ボタン → Notion Automations → Make.com → Cloud Run (本サービス) → Claude API
                                                                              ↓
                                                                       Notion レポートページ作成
```

---

## 対象ユーザー

- Notion でタスク管理とデイリーノートを運用している個人
- 日々のタスク優先度の整理や振り返りを効率化したい人
- Notion × AI による作業支援を試したいエンジニア・個人開発者

---

## 主な機能

| 機能 | 説明 |
|------|------|
| 今日やるべきタスクの抽出 | Status="Doing" のタスクから期限・工数・デイリーノートを考慮し、Claude AI が最大 10 件を優先順位付き一覧で提示。<br>マルチタスクが苦手なユーザーを想定し、取り組みやすい最初の1手を提示 |
| 期限切れタスクの検出 | 進行中だが期限が過去のタスクをすべて抽出して一覧表示 |
| 体調アドバイス | 直近 14 日分のデイリーノート【健康状態】セクションを分析し、改善提案を生成 (100-200 文字) |
| タスク管理アドバイス | 工数・課題セクションを分析し、負荷分散や効率化の提案を生成 (100-200 文字) |

---

## セットアップ

### 環境変数

`.env` ファイルを作成し、以下の変数を設定してください。

```env
SECRET_TOKEN=         # 認証トークン (Make.com からのリクエスト検証)
ANTHROPIC_API_KEY=    # Claude API キー
NOTION_TOKEN=         # Notion Integration トークン
NOTION_TASK_DB_ID=    # タスクDB の ID
NOTION_DAILY_NOTE_DB_ID=  # デイリーノートDB の ID
NOTION_REPORT_PAGE_ID=    # レポート出力先ページの ID
```

### インストール・起動

```bash
npm install
npm run build
npm start
```

### テスト

```bash
# ユニットテスト
npm test

# e2e テスト（Notion / Claude API クレジットを消費するため手動実行)
npm run test:e2e
```
