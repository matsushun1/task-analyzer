# 要件 - Notion Daily Report 自動生成システム

**作成日**: 2026-02-22
**更新日**: 2026-02-23
**対象**: 個人開発・品質優先構成

---

## 1. システム概要

### 1.1 目的
Notionで管理している進行中のタスク（Status="Doing"）と振り返りデータ（デイリーノート）を分析し、Claude AIによる優先順位付けとアドバイスを含む日次レポートを自動生成する。

### 1.2 アーキテクチャ
```
Notionボタン → Make.com → Cloud Run (TypeScript) → Claude API
                              ↓
                         Notionレポートページ作成
```

### 1.3 処理フロー
1. ユーザーがNotionのボタンを押下
2. Notion Automations が Make.com に Webhook 送信
3. Make.com が Notion API でタスクDB（Status="Doing"のみ）とデイリーノートDBを取得
4. Make.com が Cloud Run に POST リクエスト
5. Cloud Run が即座に 202 Accepted を返却（非同期処理開始）
6. Cloud Run が各ページのブロック（本文）を取得し、期限・詳細情報・工数を抽出
7. Cloud Run が Claude API を呼び出してタスク分析・アドバイス生成
8. Cloud Run が Notion API でレポートページを作成

---

## 2. 機能要件

### 2.1 コア機能

#### F-01: 今日やるべきタスクの抽出と優先順位付け
- **内容**: Status="Doing" のタスクの中から、期限が今日または過去のタスクを優先的に抽出し、Claude AIが優先順位を判断
- **データソース**:
  - タスクDB（Status="Doing" のみ）
  - タスクページのブロックから抽出した期限情報
  - デイリーノートの「翌営業日に行うこと」セクション
- **出力件数**: 最大10件
- **ソート順**: 優先度順（Claude AIが判断）

#### F-02: 期限切れタスクの抽出
- **内容**: Status="Doing" のタスクの中から、期限が過去のタスクをすべて抽出
- **対象**: 進行中だが期限を過ぎているタスク
- **期限情報の取得**: タスクページのブロックから「期限：MM/DD」形式で抽出

#### F-03: 体調に基づくアドバイス生成
- **内容**: デイリーノートの【健康状態】セクションを分析し、改善提案を生成
- **参照期間**: 直近14日分のデイリーノート
- **出力形式**: 100-200文字程度のテキスト

#### F-04: タスク管理アドバイス生成
- **内容**: タスクの進捗状況や傾向を分析し、管理方法を助言
- **参照データ**:
  - タスクの完了状況
  - デイリーノートの【課題・懸念事項】セクション
  - 工数情報（1日の合計工数、タスクごとの工数）
- **分析観点**:
  - 1日の工数が過多な場合は負荷分散を提案
  - 特定タスクに時間がかかりすぎている場合は効率化を提案
- **出力形式**: 100-200文字程度のテキスト

#### F-05: レポートページの自動生成
- **内容**: 上記F-01〜F-04の結果をNotionページとして出力
- **出力先**: 事前に指定したNotionページID

### 2.2 非機能要件

#### NF-01: 非同期処理
- Cloud Runは202 Acceptedを即座に返却し、バックグラウンドで処理を継続

#### NF-02: コスト管理
- Claude API: claude-sonnet-4-6 を使用（2026年2月最新、分析品質優先）
- Cloud Run: min-instances=0, max-instances=1（無料枠内）
- Make.com: 無料枠（月1,000オペレーション）内で運用

#### NF-03: 処理時間
- 目標: 3分以内にレポート生成完了（努力目標）

## 3. データ要件

### 3.1 タスクDB (Task List)

#### プロパティ

| プロパティ名 | Notion型 | 必須 | 説明 | 例 |
|------------|---------|------|------|-----|
| Name | Title | ✅ | タスク名 | "複数同時予約同期問題対応" |
| Status | Select | ✅ | 進捗状態 | "Doing" |
| Date Created | Created Time | ✅ | 作成日時 | 2026-02-20T02:48:00.000Z |

**Status の選択肢:**
- `To Do` (赤)
- `Doing` (黄) ← **取得対象**
- `Done 🙌` (緑)
- `アーカイブ` (オレンジ)

#### ブロック構造（本文）

タスクページの本文には以下の形式で情報を記載：

```
[] [サブタスク名]
  • 期限：2/20
  • 詳細（toggle形式）
    • [詳細内容1]
    • [詳細内容2]
```

**ブロック仕様:**
- **ルート階層のブロック**: typeは `to_do`
  - `to_do.checked = true` のブロックは完了済みとみなし、分析対象から除外する
- **期限ブロック**: typeは `bulleted_list_item`
- **詳細ブロック**: typeは `toggle` または `bulleted_list_item`
- **無視するブロック**: typeが `code`, `image`, `video`, `file`, `audio` のブロックはスキップする

**抽出対象:**
- **期限**: `期限：MM/DD` または `期限：〇月下旬` 形式
- **詳細**: 「詳細」セクション配下の箇条書き

#### 取得条件
- **フィルタ**: `Status = "Doing"`
- **理由**: 実際に着手しているタスクに絞って優先順位を判断

---

### 3.2 デイリーノートDB (Daily Notes)

#### プロパティ

| プロパティ名 | Notion型 | 必須 | 説明 | 例 |
|------------|---------|------|------|-----|
| 日付 | Date | ✅ | デイリーノートの日付 | 2026-02-22 |

#### ブロック構造（本文）

各ページの本文には以下のセクションを含む：

```
【今日行ったこと】
・デイリースクラム
・発表準備（3H）
・2.0.1
　・予約登録APIの開発（4.5H）
　　→実装、テスト仕様書作成

【翌営業日に行うこと】
・デイリースクラム
・2.0.1
　・予約登録APIの開発
　　・実装PRレビュー依頼、テスト実施方針説明

【課題・懸念事項】
・来週のテスト期間が短いこと

【健康状態】
良好
```

**抽出対象:**
- **今日行ったこと**: 完了したタスク（工数情報 `（3H）` を含む場合あり）
- **翌営業日に行うこと**: 計画中のタスク
- **課題・懸念事項**: タスク管理上の課題
- **健康状態**: 体調に関する記述

#### 取得条件
- **フィルタ**: `日付 >= (今日 - 14日)`
- **理由**: 直近2週間分で十分な傾向分析が可能

### 4.1 ページ構成

```

## 🎯 今日やるべきタスク

1. [タスク名] (期限: 2/20)
   理由: 期限が2日過ぎています

2. [タスク名] (期限: 2月下旬)
   理由: 優先度が高く、本日が期限です

... (最大10件)

━━━━━━━━━━━━━━━━━━━━━━━━

## ⚠️ 期限切れタスク

• [タスク名] - 期限: 2/15
• [タスク名] - 期限: 2/18

━━━━━━━━━━━━━━━━━━━━━━━━

## 💪 体調アドバイス

[Claude APIが生成したアドバイス]

━━━━━━━━━━━━━━━━━━━━━━━━

## 📝 タスク管理アドバイス

[Claude APIが生成したアドバイス]

━━━━━━━━━━━━━━━━━━━━━━━━

生成日時: 2026/02/22 09:30
```

### 4.2 AI分析の要件

**Claude API モデル**: claude-sonnet-4-6-20260217（2026年2月17日リリース）

**分析内容:**
1. **今日のタスク**: 期限・工数傾向を考慮した優先順位付け
2. **期限切れタスク**: 進行中だが期限超過しているタスクの抽出
3. **体調アドバイス**: 直近14日間の健康状態の傾向分析
4. **タスク管理アドバイス**: 工数・課題を踏まえた管理方法の助言
